## Findings Before Implementation

- `package_builder/packages/mxboard/elements/settings.php` declares the mxBoard system settings.
- The package builder writes settings with namespace `mxboard`; this part looks correct.
- MODX 3 `modSystemSetting::listSettings()` joins lexicon entries by exact keys:
  `setting_` + full setting key and `setting_` + full setting key + `_desc`.
- For `mxboard.task_num_format`, MODX expects:
  `setting_mxboard.task_num_format`
  and `setting_mxboard.task_num_format_desc`.
- mxBoard `ru/en/default.inc.php` currently lacks these `setting_mxboard.*` keys.
- MODX settings areas use `area_` + area key, so mxBoard should also provide:
  `area_mxboard_main`, `area_mxboard_api`, `area_mxboard_rights`, `area_mxboard_ai`.
