# PLAN: переработка карточки задачи (двухколоночная + чат + файлы)

> Согласовано с пользователем 2026-07-18. Референсы дизайна — `refs/board.png`
> (доска, «в целом норм») и `refs/task.png` (карточка задачи — образец, к которому
> идём: слева описание+мета, справа чат задачи). Реализуется **тремя подзадачами
> в отдельных сессиях** ради компактного контекста. Каждая — свой коммит + саммари
> в БЗ (`knowledge-base/tasks/`).

## Зафиксированные решения

- **Раскладка карточки** — две колонки: слева описание задачи + мета-информация,
  справа чат задачи. На узком экране колонки стекаются.
- **Файлы крепятся И к сообщениям чата (комментам), И к самой задаче.**
- **Хранение — отдельный MODX media source** (не дефолтный сайтовый). Настройка
  `mxboard.media_source` указывает на него.
- **Реакции и видеозвонки НЕ делаем** — признано оверхедом.
- **Правка/удаление комментов уже готовы** (процессоры `Task/CommentUpdate`,
  `Task/CommentDelete` + UI) — в редизайне перевязать на чат-вид, заново не писать.
- Поле типа `file` (сейчас сломано — `TypeFields.vue` рендерит его как текст) чиним
  в **подзадаче C** (та же загрузка).

## Порядок

**A и B независимы** (можно в любом порядке/параллельно). **C — после A и B.**
Пользователь начнёт с **A**.

## Подзадача A — файловая подсистема (бэкенд). НЕЗАВИСИМА

Цель: хранить и отдавать вложения задач и комментов через выделенный media source.
Никакого UI.

1. **Отдельный media source.** Создать выделенный источник файлов MODX для mxBoard
   (напр. filesystem-источник с корнем в отдельной папке загрузок, не общий сайтовый).
   Способ: резолвер пакета создаёт источник при установке (если ещё нет) и проставляет
   его id в `mxboard.media_source`; либо, если создание источника резолвером окажется
   хрупким — задокументировать ручную настройку и в резолвере только не перетирать.
   Решить в начале A. Доступ: файлы должен видеть тот, кто видит задачу (см. п.4).
2. **Модель+таблица `mxboard_attachment`** (схема `mxboard.mysql.schema.xml` +
   map-файлы `src/Model/mysql/` + `src/Model/` + миграция ADD в резолвере):
   - `task_id` (int, index) — к какой задаче;
   - `comment_id` (int, default 0, index) — >0: вложение сообщения; 0: вложение задачи;
   - `user_id` (int) — кто загрузил;
   - `name` (varchar 255) — исходное имя;
   - `path` (varchar 500) — путь в источнике;
   - `url` (varchar 500) — URL для скачивания/превью;
   - `size` (int) — байты; `ext` (varchar 20); `mime` (varchar 100, опц.);
   - `createdon` (int).
   Aggregate к `MxBoardTask`, composite от задачи (каскадное удаление вложений с задачей).
   **Камень (из БЗ):** карту миграций в `resolvers/01.resolve.tables.php` держать на
   FQCN-ключах (`MxBoardAttachment::class`), НЕ на имени таблицы — `str_replace+ucfirst`
   не разворачивает многословные имена.
3. **Процессор `Task/Upload`** (`ServiceProcessor`): принимает multipart `$_FILES` +
   `task_id` (+ опц. `comment_id`), проверяет видимость задачи (`Visibility::canView`),
   грузит файл в media source (`modMediaSource::uploadObjectsToContainer` / MODX3 API —
   уточнить методы по индексу `~/projects/apps/modx3/core/src` через codegraph), создаёт
   запись `mxboard_attachment`, возвращает её (id, name, url, size, ext, is_image).
   Лимиты (размер, расширения) — из настроек или разумные дефолты. Файлы в подпапку
   `task-<id>/`.
4. **Процессор `Task/AttachmentRemove`**: удаляет файл из источника + запись.
   Право: автор коммента (для вложения сообщения) / автор задачи / менеджер.
5. **Чтение:** `BoardQuery::taskDetail` дополнить списком вложений задачи и вложений
   по каждому комментарию (или отдельный метод `attachments(taskId)`), чтобы фронт (B/C)
   мог их показать. Отдавать `is_image` для выбора превью/чипа.
6. **Смоук** `_attach_verify_remote.php` на стенде: загрузка файла к задаче и к комменту,
   чтение через taskDetail, удаление, права. Ожидание — все OK.

Настройки/лексиконы: ключи ошибок загрузки (тип/размер/право), при необходимости
настройки лимитов (`mxboard.upload_max_size`, `mxboard.upload_extensions`).

## Подзадача B — двухколоночная сетка + чат-UI (фронт). НЕЗАВИСИМА

Цель: `TaskPage.vue` → две колонки, комментарии как чат. Файлов НЕ касается (их
вплетёт C). Эстетика — через навык `frontend-design`.

- **Слева** (скролл): мета-карточка (постановщик · исполнитель · крайний срок +
  просрочка · статус/стадия · проект · ID с копированием — как в `refs/task.png`),
  описание (ToR, markdown), поля типа, блок вердикта ИИ (уже есть), подзадачи,
  журнал (сворачиваемый). Смена стадии и дедлайн/оспаривание — тоже слева.
- **Справа** (на всю высоту, свой скролл): чат — комментарии в виде сообщений
  (аватар, имя, время, группировка по автору подряд, «свои»/«чужие» выравнивание,
  hover-действия правка/удаление — бэкенд готов). Внизу — композер (поле ввода;
  место под прикрепление файла зарезервировать, но саму загрузку включает C).
- Адаптив: < ~1000px колонки стекаются (чат под мета).
- Разнести текущий одностолбцовый шаблон аккуратно, ничего из логики не потеряв
  (moveTo, dispute/resolve, edit задачи, подзадачи, удаление).
- **Обязательно** `npm run build` (vite) перед `_deploy.sh` — иначе `board.min.js`
  не обновится. Строки — только через лексиконы (правило пользователя), новые ключи
  в `lexicon/{ru,en}`.

## Подзадача C — прикрепление файлов во фронте + починка поля `file`. ЗАВИСИТ ОТ A и B

- **Композер чата:** прикрепление файлов к сообщению (кнопка/зона), отправка через
  raw `fetch` multipart на коннектор компонента с токеном из `window.MxBoardConfig`
  (`useApi` файлы не шлёт — см. `api/connector.js`). После загрузки — привязка
  вложений к создаваемому/существующему комменту.
- **Вложения задачи:** блок «Файлы задачи» в левой колонке — загрузка/показ/удаление
  (comment_id=0).
- **Показ вложений** в сообщениях и в блоке задачи: превью для картинок
  (`is_image`), чип с иконкой + имя + размер + скачать — для прочего. Удаление —
  по праву (тост при отказе).
- **Починка поля типа `file`** в `TypeFields.vue`: сейчас проваливается в текст;
  сделать реальную загрузку через тот же механизм (значение поля — ссылка/идентификатор
  вложения). Согласовать формат хранения в `task.fields` (напр. url или attachment id).

## Общие заметки для любой сессии

- Версия/сборка: `~/.config/composer/vendor/bin/modxapp build mxboard --no-check`;
  деплой на стенд — `./_deploy.sh` (сборка+заливка+установка+сброс OPcache).
  Стенд: `hostland:modx3.art-sites.ru/htdocs/www`, PHP `/usr/local/php/php-8.3/bin/php`.
- Секреты/ключи (напр. mimo) — в `config/mimo.json` (gitignored), на стенде — в
  системных настройках.
- БЗ проекта: `knowledge-base/modx-modules/mxboard/README.md` +
  `knowledge-base/tasks/2026-07-18-mxboard-ai-check.md` (там же подводные камни:
  миграция FQCN, vite перед деплоем, ALTER TABLE до createObjectContainer).
- MODX 3 media source API уточнять по индексу `~/projects/apps/modx3/core/src`
  (`codegraph_explore ... projectPath=~/projects/apps/modx3/core/src`).
- После каждой подзадачи: коммит + пуш + саммари в `knowledge-base/tasks/` +
  обновить «Состояние» в `modx-modules/mxboard/README.md`.
