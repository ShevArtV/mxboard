# mxBoard — рабочая схема агентов (Codex-менеджер + Claude-исполнитель, MCP)

Две роли, обе подключены к доске нативно по MCP (Streamable HTTP, Bearer-токен).
Постоянной привязки «сессия ↔ задача» нет: какую карточку брать, говоришь первым
сообщением сессии.

| Роль | Личность на доске | Где запускается | MCP-токен |
|---|---|---|---|
| Менеджер (ставит/назначает/закрывает) | **Codex** (`codex`, #54) | codex CLI | в `~/.codex/config.toml` → `[mcp_servers.mxboard]` (inline header) |
| Исполнитель (делает) | **Claude** (`claude-agent`, #55) | Claude Code | user-scope (`~/.claude.json`) |

Токены-первоисточник: `config/mxboard-agents.json` (gitignored). Токен = пользователь
MODX; в БД только sha256-хэш. Потеря токена → перевыпуск provision-скриптом.

## Менеджер (Codex)
В codex-сессии доступны MCP-инструменты mxBoard (create/assign, доска, перемещение,
удаление). Сценарий: создать задачу и назначить на исполнителя Claude (#55), затем
сообщить оператору номер задачи. Закрытие (в `done`) — тоже за менеджером (он автор).
У Codex ещё подключён `chrome-devtools` MCP (headless) для браузерных проверок.

## Исполнитель (Claude)
Запускаешь сессию Claude Code (терминал/топик), первым сообщением:
> «Ты Claude, бери задачу N и делай.»

Агент через MCP-инструменты mxBoard:
1. читает карточку N (описание, поля, вложения),
2. работает в нужном репозитории,
3. двигает `in_progress` → `review`, отчитывается комментарием.

Поток стадий: `backlog → in_progress → review → done`. Исполнитель доводит максимум
до **review**; в **done** переводит только автор (менеджер) — это проверка на сервере,
не соглашение.

### Параллельные сессии
Несколько сессий Claude под одним токеном работают параллельно **по разным задачам**:
каждой называешь свой номер («задача N» / «задача M»). Одну карточку двум сессиям
одновременно не давать (нет блокировки — гонка last-write-wins). Все действия в
журнале идут от единой личности Claude (пер-агентной атрибуции нет).

## Запасной канал — REST
Тот же сервисный слой доступен по REST (`assets/components/mxboard/rest.php`, Bearer):
`GET ?path=board&project=<key>&mine=1`, `GET ?path=tasks/<id>`,
`POST ?path=tasks/<id>/move|comment`, менеджер — `POST ?path=tasks` и `DELETE ?path=tasks/<id>`.
Годится для скриптов и агентов без MCP.
