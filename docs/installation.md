# Установка

## Требования

- **MODX Revolution 3.x** (xPDO 3), контекст менеджера.
- **PHP 8.1+**, MySQL/MariaDB (InnoDB).
- **[VueTools](https://github.com/modx-pro/VueTools)** — доска использует общий Vue 3 + PrimeVue, подключаемый через Import Map этого пакета (тема Aura и PrimeIcons приходят оттуда же). Без VueTools доска покажет понятное сообщение и не запустится. Установите VueTools до или сразу после mxBoard.

## Установка пакета

Поставьте transport-пакет через **Пакеты → Установить пакет** в менеджере MODX (или из modstore, когда пакет будет опубликован). Установка идемпотентна — повторная установка/upgrade не затирает то, что вы могли донастроить руками.

## Что создаётся при установке

Резолверы пакета делают систему пригодной к работе «из коробки»:

| Что | Подробности |
|---|---|
| **Таблицы БД** | Все сущности mxBoard (`mxboard_*`). См. [Модель данных](permissions.md#модель-данных). |
| **Группа и отдел** | Группа пользователей MODX **`mxBoard`**, помеченная как отдел «Отдел разработки». |
| **Типы задач** | `bugfix` (поля: где, что, как воспроизвести, как должно быть) и `feature` (цель, критерии приёмки). |
| **Проект `default`** | «Основной проект» с колонками `backlog` → `in_progress` → `review` → `done`, плюс глобальный шаблон колонок (для новых проектов). |
| **Медиа-источник** | Выделенный источник файлов **«mxBoard»** для вложений; его id пишется в настройку `mxboard.media_source`. |
| **Политика доступа** | **`mxBoard Only`** — пресет «только канбан» для киоск-доступа (никому не назначается автоматически). См. [Киоск-доступ](integration.md#киоск-доступ-только-канбан). |
| **Плагины** | `mxBoardProfileToken` (виджет выдачи токена в профиле, только sudo) и `mxBoardKiosk` (редирект киоск-групп на доску). |
| **События** | 11 системных событий `mxbOn*` для интеграции. См. [Интеграция](integration.md). |

## Сборка из исходников

Живой MODX для сборки не нужен — сборка headless:

```bash
~/.config/composer/vendor/bin/modxapp build mxboard --no-check
```

## Первый запуск

Откройте **Компоненты → mxBoard** — доска проекта `default` откроется сразу. Дальше:

1. Заведите пользователей и добавьте их в группу-отдел (см. [Быстрый старт](quick-start.md)).
2. Выдайте агенту токен: **профиль пользователя → виджет «Токен агента»** (доступно sudo). Подробнее — [Агенты](agents.md).
3. При необходимости настройте [системные настройки](settings.md): ИИ-проверку полноты, лимиты вложений, киоск-доступ.

## Грабли, о которых стоит знать

- **OPcache в PHP-FPM живёт отдельно от CLI.** После установки/обновления пакета из консоли веб-сайт может продолжать выполнять предыдущую версию классов — сбросьте OPcache веб-запросом.
- **Поле порядка называется `position`, а не `rank`.** В MySQL 8 `RANK` — зарезервированное слово; xPDO не квотит его в `ORDER BY` и **молча возвращает пустую выборку**. Это заложено в схему — важно помнить при доработках.
