<strong>mxBoard: канбан-доска для команд, людей и ИИ-агентов в MODX 3</strong>

<blockquote>mxBoard превращает менеджер MODX 3 в рабочую канбан-доску для задач, исполнителей и подключенных ИИ-агентов. Человек ставит задачу, агент или внешний исполнитель берет ее в работу, пишет план, ждет подтверждения, выполняет, отчитывается и отдает результат на review.</blockquote>

<strong>Быстрый старт</strong>

<ul>
<li>Компонент ставится как обычный MODX-пакет и работает внутри менеджера.</li>
<li>Из коробки есть канбан-интерфейс на Vue 3/PrimeVue, REST API и MCP-эндпоинт для ИИ-инструментов.</li>
<li>Ядро mxBoard не привязано к конкретному боту или оркестратору: подключайте своих ИИ-агентов, внутренние скрипты и внешние системы через MCP/REST и Bearer-токены.</li>
</ul>

<strong>Зачем это нужно</strong>

<ul>
<li>ИИ-агенты уже умеют писать код, проверять гипотезы, готовить документы и разбирать задачи. В реальной работе быстро появляется другой вопрос: как контролировать качество выполнения и историю задач.</li>
<li>mxBoard дает общее состояние внутри MODX. Человек видит понятную доску в менеджере, агент видит те же задачи через MCP или REST, а все действия проходят через один серверный слой прав.</li>
<li>UI, REST и MCP используют одни и те же сервисы и правила. Агент не может найти более слабый канал и обойти ограничения.</li>
</ul>

<strong>Канбан в менеджере MODX</strong>

<ul>
<li>Главный интерфейс mxBoard — канбан-доска в менеджере MODX.</li>
<li>Задачи разложены по проектам и колонкам, карточки можно двигать по стадиям, открывать в детальном виде, обсуждать ход работы и смотреть историю переходов.</li>
<li>Иерархия простая: отдел — группа пользователей MODX; проект — отдельная доска отдела; задача — типизированная карточка с автором, исполнителем, дедлайном, полями, комментариями и вложениями.</li>
<li>Для команд это рабочее место прямо там, где уже живет сайт и его пользователи. Для агентских сценариев это единый источник правды для человека и автоматизации.</li>
</ul>

<strong>Настраиваемые типы задач и поля</strong>

<ul>
<li>В mxBoard нет абстрактной задачи без структуры. Каждая задача относится к типу, а тип задает набор полей, которые нужно заполнить при постановке.</li>
<li>Для багфикса можно потребовать URL, фактическое поведение, ожидаемое поведение, шаги воспроизведения и артефакты. Для новой функции — цель, пользовательский сценарий, ограничения, критерии приемки и материалы.</li>
<li>Доступны текстовые поля, многострочный текст, URL, числа, даты, пользователь и файловые поля.</li>
<li>Агент может получить схему типа через MCP или REST перед созданием задачи и понять, какие данные обязательны. Вместо размытого "почини форму" постановка становится проверяемым набором полей.</li>
</ul>

<strong>ИИ-валидатор полноты постановки</strong>

<ul>
<li>Для каждого типа задачи можно включить ИИ-проверку полноты.</li>
<li>Перед сохранением mxBoard отправляет заголовок, постановку и заполненные поля в OpenAI-совместимый провайдер и получает вердикт: достаточно ли данных, чего не хватает, насколько задача готова к работе.</li>
<li>Проверка работает в строгом режиме, когда неполную задачу создать нельзя, или в мягком режиме, когда пользователь видит предупреждение и может осознанно продолжить.</li>
<li>Если ИИ-провайдер временно недоступен, доска не блокируется: задача пропускается, а проблема фиксируется в журнале.</li>
</ul>

<strong>MCP, REST и Bearer-токены для агентов</strong>

<ul>
<li>MCP-эндпоинт: <em>assets/components/mxboard/mcp.php</em>.</li>
<li>REST-эндпоинт: <em>assets/components/mxboard/rest.php</em>.</li>
<li>Авторизация идет через Bearer-токены. Токен выпускается в профиле пользователя MODX в виджете "Токен агента", показывается один раз, а в базе хранится только sha256-хэш.</li>
<li>Права агента равны правам пользователя, которому выдан токен.</li>
<li>Через MCP доступны инструменты для списка проектов, схем типов, пользователей отдела, создания задач, просмотра карточки, комментариев, перевода по колонкам, оспаривания дедлайна и других рабочих действий.</li>
<li>Структурные инструменты — создание отделов, проектов и типов задач — видит только менеджер отдела.</li>
</ul>

<strong>Роли автора и исполнителя</strong>

<ul>
<li>У каждой задачи есть автор и исполнитель. Это не просто подписи на карточке, а основа модели прав.</li>
<li>Автор ставит задачу, принимает результат и закрывает карточку.</li>
<li>Исполнитель берет задачу в работу, двигает ее по рабочим стадиям и отчитывается.</li>
<li>Колонки могут ограничивать, кто имеет право перевести задачу в конкретную стадию: автор, исполнитель, или оба.</li>
<li>Финальная колонка защищена отдельно: закрытие разрешено автору или менеджеру.</li>
<li>Есть запрет самоаттестации: если автор и исполнитель совпадают, задача не может быть закрыта самим собой, пока это явно не разрешено настройкой.</li>
</ul>

<strong>Контроль выполнения и review</strong>

<ol>
<li>Постановщик создает типизированную задачу и назначает исполнителя.</li>
<li>Агент получает задачу через MCP/REST, читает схему и карточку.</li>
<li>Агент пишет план в комментарий и ждет подтверждения.</li>
<li>После согласования агент выполняет работу, оставляет отчет и переводит задачу на review.</li>
<li>Автор проверяет результат и закрывает задачу.</li>
</ol>

<ul>
<li>Если для выполнения нужна помощь другого участника, создаются подзадачи.</li>
<li>Незавершенные подзадачи блокируют закрытие родительской задачи, поэтому результат нельзя случайно принять, пока зависимая работа еще открыта.</li>
</ul>

<strong>Чат, комментарии и вложения</strong>

<ul>
<li>У задачи есть полноценный чат. Комментарии подходят для планов, промежуточных отчетов, вопросов, решений и ревью.</li>
<li>Свои комментарии можно редактировать и удалять в рамках прав.</li>
<li>К задачам и сообщениям можно прикреплять файлы.</li>
<li>Вложения хранятся через выделенный media source MODX, поддерживаются лимиты размера, количества и расширений.</li>
<li>Файлы задачи и файлы сообщений разделены, а при удалении комментария или задачи связанные физические файлы очищаются каскадно.</li>
</ul>

<strong>SSE и in-app уведомления</strong>

<ul>
<li>mxBoard умеет показывать живые уведомления прямо в менеджере MODX: колокольчик, счетчик и всплывающие сообщения.</li>
<li>SSE-эндпоинт работает через сессию менеджера и автоматически переподключается.</li>
<li>Жизненные события задачи материализуются в очередь уведомлений для участников: автора и исполнителя.</li>
<li>Внешние интеграции могут использовать системные события MODX или REST-ленту <em>/events</em>, если им нужен собственный оркестратор, webhook или polling.</li>
</ul>

<strong>Универсальное ядро без привязки к конкретному боту</strong>

<ul>
<li>mxBoard не зашивает внутрь логику конкретного мессенджера, агента или трекера.</li>
<li>Для интеграций есть системные события <em>mxbOnTaskCreate</em>, <em>mxbOnTaskMove</em>, <em>mxbOnTaskClose</em>, <em>mxbOnTaskComment</em>, <em>mxbOnTaskUpdate</em>, <em>mxbOnTaskDelete</em> и другие.</li>
<li>У карточки есть JSON-поле <em>meta</em>. В нем можно хранить рабочий каталог, внешний идентификатор, тему в мессенджере, параметры запуска агента или любые данные вашей интеграции.</li>
<li>Ядро mxBoard эти данные не интерпретирует, поэтому компонент остается универсальным.</li>
</ul>

<strong>Для кого</strong>

В первую очередь пакет предназначен для персонального использования. Если вы уже пользуетесь CLI агентами данный инструмент будет вам полезен для улучшения качества вывполнения задач, оптимизации расходов, лучшего контроля.
Так же можно использовать и для обычной командной работы как простой и быстрый вариант канбан-доски разворачиваемый как обычный сайт на любом шареде со всеми плюшками Modx.

<strong>Технические требования</strong>

<ul>
<li>MODX Revolution 3.x.</li>
<li>PHP 8.1+.</li>
<li>MySQL/MariaDB с InnoDB.</li>
<li>VueTools для интерфейса на Vue 3 и PrimeVue.</li>
<li>Лицензия проекта — MIT.</li>
</ul>

<blockquote>mxBoard находится в alpha-статусе, но уже включает основной рабочий контур: канбан в менеджере, типы задач, ИИ-проверку, MCP/REST, токены агентов, роли автора и исполнителя, комментарии, вложения, журнал и SSE-уведомления.</blockquote>

<strong>Коротко</strong>

<ul>
<li>mxBoard — это канбан для MODX 3, сделанный с учетом новой реальности: задачи выполняют не только люди, но и агенты.</li>
<li>Компонент дает структуру постановки, серверные правила, подключение через MCP/REST и прозрачный review-процесс, где человек остается владельцем приемки, а автоматизация работает в заданных границах.</li>
</ul>
